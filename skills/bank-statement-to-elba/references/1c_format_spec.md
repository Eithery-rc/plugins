# 1CClientBankExchange для Эльбы

Что экспериментально подтверждено апрелем 2026 (см. sandbox `Nalogi/scratch_1c_test.py`):

## Работает
- UTF-8 кодировка, даже если в шапке `Кодировка=Windows`.
- `\r\n` переводы строк.
- `ВерсияФормата=1.03`.
- Placeholder `РасчСчет` (Эльба сама создаёт счёт в «Реквизитах» при первой загрузке, определяя банк по БИК).
- Настоящий `ПолучательИНН` + `Получатель=<ФИО>` → warning «проверьте организацию» не появляется.
- `СекцияДокумент=Банковский ордер` принимается, отображается как «п.п.» в Эльбе (косметика, не критично).
- `ВидОплаты=01`, `Очередность=5`.

## Не работает / ограничения
- **Одна валюта в файле** — Эльба отвергает смеси с текстом «Нельзя загрузить файл с разными валютами».
- **Тип операции** не задаётся из файла — всегда дефолт «Оплата товаров и услуг клиентами». Меняется bulk-edit в UI после импорта.
- **Дедупликация отсутствует** — повторный импорт = дубли строк.

## Обязательные поля шапки
```
1CClientBankExchange
ВерсияФормата=1.03
Кодировка=Windows
Отправитель=<любая строка>
Получатель=<любая строка>
ДатаНачала=DD.MM.YYYY
ДатаКонца=DD.MM.YYYY
РасчСчет=<20 цифр>
```

## Секция расчётного счёта
```
СекцияРасчСчет
ДатаНачала=...
ДатаКонца=...
РасчСчет=...
НачальныйОстаток=0.00
ВсегоПоступило=<сумма>
ВсегоСписано=0.00
КонечныйОстаток=<сумма>
КонецРасчСчет
```

## Секция документа (входящее поступление)
```
СекцияДокумент=Банковский ордер
Номер=<ref из выписки>
Дата=DD.MM.YYYY
Сумма=<decimal 2>
Плательщик=<контрагент>
Плательщик1=<контрагент>
ПлательщикСчет=<20 цифр>
ПлательщикИНН=<10/12 цифр>
ПлательщикБИК=<9 цифр>
ПлательщикБанк1=<строка>
ПлательщикКорсчет=<20 цифр>
Получатель=<ФИО ИП>
Получатель1=<ФИО ИП>
ПолучательСчет=<20 цифр>
ПолучательИНН=<12 цифр>
ПолучательБИК=<9 цифр>
ПолучательБанк1=<строка>
ПолучательКорсчет=<20 цифр>
ВидОплаты=01
Очередность=5
НазначениеПлатежа=<строка>
ДатаПоступило=DD.MM.YYYY
КонецДокумента
```
